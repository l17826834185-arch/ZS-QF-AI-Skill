# 产品对比说明

用途：
- 面向客户回答“智书AI合同和其他合同、电子签、法律 AI 产品有什么区别”。
- 以客观维度解释差异，不贬低其他厂商。

## 回答原则

- 先判断客户关注的是签署、合同全生命周期、AI 审查、办公生态，还是法律大模型能力。
- 使用“适合场景不同”来解释差异，避免使用“谁更强、谁更弱”的表达。
- 涉及第三方产品能力、价格、部署方式和版本权益时，应以对方官方最新资料为准。
- 如果信息不确定，应明确说明“需要以该产品当前版本为准”。

## 电子签 / 合同协同类产品

代表产品：
- e签宝
- 法大大
- 契约锁

常见关注点：
- 电子签署、签章、签署流程、印章管理、签署合规。
- 部分产品也提供合同协同、合同管理、AI 审查或智能起草能力。

建议回答：

> e签宝、法大大、契约锁等产品通常更容易从电子签署、签章合规和签署流程角度被客户理解。智书AI合同更强调飞书内的合同全生命周期管理，覆盖起草、协商、审查、审批、比对、归档、履约和数据分析。  
> 如果您主要关注“如何签署和盖章”，可以重点了解电子签产品；如果您希望把合同管理放进飞书日常工作流，并关注合同审查、归档、履约和数据沉淀，可以进一步了解智书AI合同。

## AI 合同审查工具

代表产品：
- 幂律智能 MeCheck
- 法天使智能审查
- 案牍AI

常见关注点：
- 合同条款审查、风险识别、修改建议、审查清单、合同模板或法律知识能力。

建议回答：

> 独立 AI 合同审查工具通常更聚焦单次合同审查体验，例如风险识别、审查清单和修改建议。智审平台同样聚焦 AI 合同审查，而智书AI合同则进一步覆盖合同起草、审批、归档、履约和数据分析等完整管理链路。  
> 如果您只需要独立审查工具，可以重点比较审查规则、文档体验和适用合同类型；如果希望审查结果继续进入飞书审批、归档、履约和数据管理流程，可以了解智书AI合同或智审平台。

## 办公平台生态产品

代表产品：
- 钉钉智能合同
- 其他飞书生态合同应用

常见关注点：
- 办公平台入口统一。
- 与组织、审批、消息、文档、表格等能力集成。

建议回答：

> 如果企业已经深度使用某个办公平台，合同产品与办公平台的集成深度会直接影响使用体验。智书企飞的特点是围绕飞书生态做产品和服务，支持结合飞书组织、审批、消息、群聊、文档、多维表格等场景开展合同管理和业务流程管理。

## 法律大模型 / API 类产品

代表产品：
- 通义法睿
- 其他法律大模型 API 或模型服务

常见关注点：
- 法律问答、文本生成、文书处理、API 调用、模型能力输出。

建议回答：

> 法律大模型或 API 更偏底层能力，适合有研发能力并希望自行搭建业务系统的客户。智书AI合同和智审平台更偏具体业务应用，把 AI 能力放到合同审查、审批、归档、履约和数据管理流程中。  
> 如果您只需要模型接口，可以重点了解法律大模型 API；如果希望直接使用合同管理和合同审查应用，可以了解智书AI合同或智审平台。

## 简短回答模板

> 这几类产品关注点不同：电子签产品更偏签署和签章合规，AI 审查工具更偏合同风险识别，法律大模型更偏底层模型能力。智书AI合同更强调飞书内的合同全生命周期管理，把起草、协商、审查、审批、比对、归档、履约和数据分析放在同一条流程里。具体选择建议结合您当前最关注的场景判断。
